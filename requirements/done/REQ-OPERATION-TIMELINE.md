# REQ-OPERATION-TIMELINE — Codex 式操作级时间线

> **状态**：已实施（待联调验收）  
> **优先级**：P1（体验增强，不阻塞阶段二认证）  
> **关联**：[`2026-06-13-processing-timeline-v2-design.md`](../../docs/superpowers/specs/2026-06-13-processing-timeline-v2-design.md)（V2 已落地）

## 背景

当前对话页将「处理过程」「思考过程」「最终回答」拆成三个独立区域：步骤只有三态摘要文案，思考汇总在消息级 `reasoning` 字段，用户需多次展开才能对齐「某一步里模型在想什么、产出了什么」。刷新后虽可恢复，但无法像 Codex 一样**按操作逐步展示：思考（流式）→ 中间输出 → 结果 + 耗时**。

## 目标

1. **操作级卡片**：每个步骤（意图识别、检索、工具调用、Agent 推理、生成回答）独立卡片，可展开/折叠。
2. **步骤内流式思考**：思考内容绑定到具体 `stepId`，SSE 增量推送，执行中可见（默认不隐藏）。
3. **步骤内输出与结果**：检索命中、工具返回等作为步骤 `output`/`result`，与最终 Markdown 回答区分。
4. **耗时与状态**：每步 `durationMs`；running 时水波纹/脉冲动效。
5. **事件全触达**：所有 delta 经 SSE → Redis 重连缓冲 → 落库，刷新可重放。
6. **向后兼容**：旧客户端可继续解析 V2 `type:step`；新字段可选；遵循设计文档 §3.5 扩展规范

## 非目标

- 运维 Trace / SkyWalking 联动
- 知识库检索结果全文内嵌（MVP 仍用摘要 + 可选折叠 output）
- 多语言 i18n 配置化
- 修改 LLM Gateway / 外部厂商 API

## 成功标准

| # | 验收项 |
|---|--------|
| 1 | 知识库类问题：intent → rag → agent → generate 四卡可见，rag 卡内可见流式思考（若有）与「命中 N 条」结果 |
| 2 | 简单对话：intent → generate 两卡，generate 卡内可见正文流式输出 |
| 3 | 流式生成中 F5：重连后各卡状态与已输出思考/正文一致 |
| 4 | 完成后刷新：从 DB 恢复 steps JSON（含 reasoning/output）+ content，卡片可展开 |
| 5 | E2E：`processing-timeline-real.spec.ts` 扩展通过 |

## 用户故事

- 作为使用者，我希望看到 AI **正在做哪一步**，以及该步**实时思考**，而不是等全部完成后才看到一段总思考。
- 作为使用者，我希望每一步有**耗时**，方便判断瓶颈在检索还是生成。
