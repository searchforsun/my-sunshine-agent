# 阶段 1.6 — Redis SSE 无感重连（Track G）+ Track F 收尾

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** LLM 生成与 SSE 推送解耦；客户端断线/刷新后通过 Redis Stream 自动追读；generation 已停时 410 降级 Track F「继续生成」。

**Architecture:** `GenerationJob` 独立订阅 LLM Flux，chunk 写入 Redis Stream + MySQL partial flush；SSE 端点订阅 Redis 而非直接订阅 LLM。`GET /chat/stream/{generationId}?afterSeq=N` 重连追读。BFF 透传；前端 localStorage 存 active generation 指针，mount 自动 reconnect。

**Tech Stack:** JDK 21, Spring Boot 3.2.9, spring-data-redis, Redis Stream (XADD/XREAD), WebFlux SSE, Vue3

**设计 spec：** [2026-06-11-phase1.5-conversation-mvp-design.md](../specs/2026-06-11-phase1.5-conversation-mvp-design.md) — Track F（已完成）+ Track G

**前置：** Track A～E 已落地；Track F（resume API + flush + 继续生成 UI + 集成测试）已可用。

**Files:**
- Create: `orchestrator/src/main/java/com/sunshine/orchestrator/generation/GenerationJob.java`
- Create: `orchestrator/src/main/java/com/sunshine/orchestrator/generation/GenerationRegistry.java`

- [x] **Step 1:** GenerationRegistry

```java
@Component
public class GenerationRegistry {
    private final ConcurrentHashMap<String, GenerationJob> running = new ConcurrentHashMap<>();

    public GenerationJob register(GenerationJob job);
    public Optional<GenerationJob> get(String generationId);
    public void cancel(String generationId);  // dispose LLM + INTERRUPTED
    public boolean tryLockMessage(String messageId, String generationId); // 防双 Job
}
```

- [x] **Step 2:** GenerationJob — 独立 subscribe LLM Flux

```java
public class GenerationJob {
    public void start(Flux<String> llmFlux, StringBuilder mysqlBuffer,
            Consumer<String> flushPartial);
    public void onSubscriberGone();  // 启动 orphan-timeout 计时，不 dispose LLM
    public void onSubscriberAttached(); // 重置 orphan 计时
    public void cancel();            // dispose + INTERRUPTED + commitFinal partial
}
```

要点：
- `onNext`: seq++ → XADD + append mysqlBuffer + 周期性 flushPartial
- `onComplete`: status=COMPLETED + commitFinal
- `onError`: status=FAILED
- orphan-timeout 到期：cancel() 同 Stop 行为
